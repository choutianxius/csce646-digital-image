Up in the Cloud Sea

You are viewing the compiled and animated shader, running in an HTML canvas element. This is achieved by implementing a minimal rendering engine with WebGL2, following similar behavior as ShaderToy's.

Source code for this web page, as well as the rendering engine can be found in this Github repo.

ShaderToy Code

The source code can be found in this link. It is based on mdb's work on ShaderToy, by combining the code for BufferA into the Image code.

The clouds in the shader are created by sampling from a blue-noise image. When running on ShaderToy, you should set up the blue-noise texture in iChannel0. You can also choose other textures to generate different clouds.

How Is Vector to Raster Implemented

The train and the bridge are generated from pure vector-to-raster calculation. The clouds are generated by first sampling from a blue-noise image.

How is Image Manipulation Implemented

  • "Vignett correction" is applied to the generated frames so that the border regions become darker.
  • I also applied varying brightness based on iTime.
